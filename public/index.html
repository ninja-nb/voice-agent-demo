Voice Channel Orchestrator Demo

Run an end-to-end agent turn with speech input, tool grounding, model reasoning, and voice response output.

Demo Flow Overview

End-to-end agent execution pipeline: input capture -> STT adapter -> search tool grounding -> LLM provider inference -> TTS adapter playback. You can run a synchronous turn or stream orchestration events.

Quick Steps

  1. Step 1: Choose an LLM Provider and LLM Model.
  2. Step 2: Capture user utterance by recording or upload an audio file (auto-stop at 30s, with a 5s warning).
  3. Step 3: Run the turn with Run Live (streaming stages) or Run Once (single final response).
  4. Step 4: Review transcript, grounded answer, search evidence, and synthesized voice output.
  5. Step 5: Use Observability & Telemetry for stage timing, event trail, and runtime details.

Scope: Assumptions & Future Improvements

      

        This is a prototype for architecture demonstration, not a production deployment.
      

      
    
        
  • Assumption: Trusted demo environment with controlled usage and API keys.
    • 
        
  • Assumption: Single-turn request flow focused on orchestration clarity over feature depth.
    • 
        
  • Security & Access: Authentication, rate limiting, and per-user quota controls for production rollout.
    • 
        
  • Safety & Policy: Formal guardrails and policy enforcement framework integration.
    • 
        
  • Reliability Engineering: Timeouts, retries, circuit breaking, and fallback routing patterns.
